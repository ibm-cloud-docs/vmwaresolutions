# Tâches de configuration
{: #opsprocs-configure}

Une fois que votre instance vCenter Server a été mise à disposition, il existe un certain nombre de tâches de configuration que les administrateurs système peuvent avoir besoin d'exécuter pour adapter l'environnement initial aux besoins de leur entreprise, ainsi que pour répondre aux futures demandes de service. Pour faciliter l'utilisation, nous les avons classés comme suit :

* Consignes pour les instances et les clusters
* Procédures relatives aux machines virtuelles
* Procédures vCenter
* Procédures relatives aux hôtes vSphere ESXi
* Procédures de stockage
* Procédures de réseau

## Consignes pour les instances et les clusters
{: #opsprocs-configure-instance}

Tableau 1. Clusters et instances vCenter Server

| Titre | Description |
|---|---|
| Convention de dénomination  | L'une des premières tâches à accomplir est d'adopter une convention de dénomination pour votre instance vCenter Server. Vous pouvez étendre la convention de dénomination existante de votre organisation ou en créer une. Pour plus d'informations, voir [Naming Convention in VMware Validated Design 4.1](https://docs.vmware.com/en/VMware-Validated-Design/4.1/com.vmware.vvd.sddc-upgrade.doc/GUID-FCA126F1-E429-454E-B3BA-226525742DC1.html){:new_window}. Voir [Remarques relatives à la modification des artefacts vCenter Server](/docs/services/vmwaresolutions/vmonic?topic=vmware-solutions-vcenter_chg_impact#vcenter_chg_impact){:new_window} avant d'adopter une convention de dénomination et de changer les noms dans votre instance vCenter Server.|
| Connexion à votre instance vCenter Server | Une fois la mise à disposition terminée, connectez-vous à vCenter. Il existe deux méthodes pour cela au départ. Vous pouvez déployer un serveur "jumpbox" pour vous connecter à la jumpbox via Internet et ensuite de la jumpbox à vCenter via le réseau privé. La deuxième méthode consiste à utiliser le VPN {{site.data.keyword.cloud_notm}} qui est une connexion VPN de votre emplacement au réseau privé {{site.data.keyword.cloud_notm}} qui permet une gestion hors bande et un accès à vCenter. Pour plus d'informations, voir [Getting started with Virtual Private Networking (VPN)](/docs/infrastructure/iaas-vpn?topic=VPN-gettingstarted-with-virtual-private-networking){:new_window}.|
| Configuration de la résolution de nom | Pour utiliser les fonctionnalités de vCenter, ajoutez la résolution de nom des composants vCenter Server à votre station de travail ou jumpbox en fonction de votre méthode de connexion. Le fichier /etc/hosts ou c:\Windows\System32\Drivers\etc\hosts nécessite des entrées pour le PSC (Platform Services Controller), vCenter et les hôtes vSphere ESXi. Pour plus d'informations, voir [Déploiement d'un fichier OVF à l'aide du client Web VMware vSphere](/docs/services/vmwaresolutions?topic=vmware-solutions-trbl_deploy_ovf#trbl_deploy_ovf){:new_window}. |
| RBAC | Pour plus d'informations sur les privilèges requis pour les tâches courantes dans l'environnement vSphere, voir [Required Privileges for Common Tasks](https://docs.vmware.com/en/VMware-vSphere/6.7/com.vmware.vsphere.vm_admin.doc/GUID-4D0F8E63-2961-4B71-B365-BBFA24673FDB.html){:new_window}. |
| Utilisation d'un certificat personnalisé signé par une autorité de certification |  Pour plus d'informations sur le remplacement d'un certificat SSL vSphere 6.x Machine par un certificat personnalisé signé par une autorité de certification, voir [Replacing a vSphere 6.x Machine SSL certificate with a Custom Certificate Authority Signed Certificate (2112277)](https://kb.vmware.com/s/article/2112277){:new_window}. |
| Configuration de la surveillance vSphere | vSphere inclut un sous-système d'événements et d'alarmes configurables par l'utilisateur qui suit les événements des composants vSphere et stocke les données dans des fichiers journaux et dans la base de données vCenter. Les événements sont des enregistrements d'actions utilisateur ou système qui se produisent sur des objets vSphere. Il existe trois types d'événements : les informations, les avertissements et les erreurs. Les alarmes sont des notifications qui sont activées en réponse à un événement, un ensemble de conditions ou l'état d'un objet vSphere. Les niveaux de gravité des alarmes sont les suivants : Normal - vert, Avertissement - jaune et Alerte - rouge. Les actions d'alarme sont des opérations qui se produisent en réponse à une alarme, par exemple l'envoi d'une notification par e-mail. Pour plus d'informations sur la configuration et l'utilisation de la surveillance vSphere, voir [Surveillance des événements, des alarmes et actions automatiques](https://docs.vmware.com/en/VMware-vSphere/6.5/com.vmware.vsphere.monitoring.doc/GUID-9272E3B2-6A7F-427B-994C-B15FF8CADC25.html){:new_window}. |

Vous pouvez ajouter des clusters supplémentaires à vos instances pour augmenter la capacité de calcul et de stockage. Pour plus d'informations, voir [Ajout, affichage et suppression de clusters pour des instances vCenter Server](/docs/services/vmwaresolutions/services?topic=vmware-solutions-vc_addingviewingclusters#vc_addingviewingclusters){:new_window}. |
| Capacité de vCenter Server | Vous pouvez étendre ou contracter la capacité de votre instance VMware vCenter Server en fonction des besoins de votre entreprise, en ajoutant ou en supprimant des serveurs vSphere ESXi ou du stockage NFS. | Services d'instance vCenter Server | Après le déploiement de votre instance vCenter Server, vous pouvez ajouter des services supplémentaires de notre portefeuille, tels qu'une solution de reprise après incident, de gravité ou de sauvegarde. Si vous n'avez plus besoin de ces services, vous pouvez les supprimer de vos instances. Pour plus d'informations, voir [Commande, affichage et retrait de services pour des instances vCenter Server](/docs/services/vmwaresolutions?topic=vmware-solutions-vc_addingremovingservices#vc_addingremovingservices){:new_window}. |
| Mise à jour des instances vCenter Server | Pour plus d'informations sur l'application des mises à jour aux instances vCenter Server, voir [Applying IBM management component updates to vCenter Server instances](/docs/services/vmwaresolutions?topic=vmware-solutions-vc_applyingupdates#vc_applyingupdates). Pour plus d'informations sur les administrateurs système qui appliquent manuellement les mises à jour des produits VMware, voir [Présentation de vCenter Server on IBM Cloud with Hybridity Bundle](/docs/services/vmwaresolutions?topic=vmware-solutions-vcs-hybridity-intro#vcs-hybridity-intro){:new_window}. |
| Configuration multisite | Le modèle de configuration multisite utilise un concentrateur et une topologie en étoile (hub and spoke) avec un site principal et un maximum de sept sites secondaires. Pour plus d'informations sur les configurations multisite pour les instances vCenter Server on {{site.data.keyword.cloud_notm}}, voir [Configuration multisite pour des instances vCenter Server on IBM Cloud](/docs/services/vmwaresolutions?topic=vmware-solutions-vc_multisite#vc_multisite){:new_window}. Pour plus d'informations sur les étapes de suppression des instances vCenter Server dans une configuration multisite, voir [Suppression d'instances vCenter Server dans une configuration multisite](/docs/services/vmwaresolutions?topic=vmware-solutions-vc_deletinginstance_multi#vc_deletinginstance_multi){:new_window}. |

| Sauvegarde des principaux composants d'une instance vCenter Server | Vos administrateurs système sont responsables de la configuration, de la gestion et de la surveillance de tous les composants logiciels de l'instance vCenter Server, y compris la sauvegarde et la disponibilité de l'infrastructure de gestion et des charges de travail. Dans le cadre de la solution, vous pouvez déployer les services complémentaires IBM Spectrum Protect Plus on {{site.data.keyword.cloud_notm}} ou Veeam on {{site.data.keyword.cloud_notm}}. Veeam et IBM Spectrum Protect Plus peuvent vous aider à répondre aux exigences de sauvegarde relatives à vos composants de gestion. Pour plus d'informations sur les mécanismes de sauvegarde et les composants clés à sauvegarder, voir [Sauvegarde des composants](/docs/services/vmwaresolutions/archiref/solution/solution_backingup.html#vcenter-file-based-backup){:new_window}. |
| Collecte d'informations de diagnostic pour les produits VMware | Le support technique VMware demande régulièrement des informations de diagnostic lorsqu'une demande de support est traitée. Ces informations de diagnostic contiennent des journaux spécifiques au produit, des fichiers de configuration et des données appropriées à la situation.
