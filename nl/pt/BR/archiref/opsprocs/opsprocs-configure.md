# Tarefas de configuração
{: #opsprocs-configure}

Depois que a instância do vCenter Server é provisionada, há várias tarefas de configuração que os administradores do sistema podem precisar executar para customizar o ambiente inicial para as necessidades de sua empresa e para responder a futuras solicitações de serviço.

Essas tarefas de configuração são:

* Orientação de instância e cluster
* Procedimentos da máquina virtual (VM)
* Procedimentos do vCenter
* Procedimentos de host do vSphere ESXi
* Procedimentos de armazenamento
* Procedimentos de rede

## Orientação de instância e cluster
{: #opsprocs-configure-instance}

Tabela 1. Instâncias e clusters do vCenter Server

| Título | Descrição |
|---|---|
| Convenção de nomenclatura  |  Uma das primeiras tarefas é adotar uma convenção de nomenclatura para a instância do vCenter Server. É possível estender a convenção de nomenclatura existente que é usada por sua organização ou criar uma nova. Para obter mais informações, consulte [Convenção de nomenclatura no VMware Validated Design 4.1](https://docs.vmware.com/en/VMware-Validated-Design/4.1/com.vmware.vvd.sddc-upgrade.doc/GUID-FCA126F1-E429-454E-B3BA-226525742DC1.html){:new_window}. | Conectando-se à sua instância do vCenter Server  | Após o provisionamento, conecte-se ao vCenter. Inicialmente, há dois métodos para isso. É possível implementar um servidor "jumbox" para que seja possível se conectar ao jumpbox usando a Internet e, em seguida, do jumpbox ao vCenter usando a rede privada. O segundo método é usar a VPN do {{site.data.keyword.cloud_notm}}, que é uma conexão VPN de seu local com a rede do {{site.data.keyword.cloud_notm}} Private, que permite gerenciamento fora da banda e acesso ao vCenter. Para obter mais informações, veja [Introdução à Virtual Private Networking (VPN)](/docs/infrastructure/iaas-vpn?topic=VPN-getting-started). |
| Configurar resolução de nome |  Para usar os recursos no vCenter, inclua a resolução de nome para os componentes do vCenter Server em sua estação de trabalho ou jumbox, dependendo de seu método de conexão. O arquivos /etc/hosts ou c:\Windows\System32\Drivers\etc\hosts requer entradas para os hosts do Platform Services Controller (PSC), do vCenter e do vSphere ESXi. Para obter mais informações, consulte [Implementando um arquivo OVF por meio do VMware vSphere Web Client](/docs/services/vmwaresolutions?topic=vmware-solutions-trbl_deploy_ovf#trbl_deploy_ovf){:new_window}. |
| RBAC | Para obter informações sobre os privilégios necessários para tarefas comuns no ambiente do vSphere, consulte [Privilégios necessários para tarefas comuns](https://docs.vmware.com/en/VMware-vSphere/6.7/com.vmware.vsphere.vm_admin.doc/GUID-4D0F8E63-2961-4B71-B365-BBFA24673FDB.html){:new_window}. |
| Usando um certificado assinado da autoridade de certificação customizada |  Consulte [Substituindo um certificado SSL da máquina do vSphere 6.x por um Certificado assinado de autoridade de certificação customizada (2112277)](https://kb.vmware.com/s/article/2112277){:new_window}. |
| Configurar o monitoramento do vSphere  | O vSphere inclui eventos configuráveis pelo usuário e subsistema de alarmes, que rastreia eventos dos componentes do vSphere e armazena os dados em arquivos de log e no banco de dados do vCenter. Eventos são registros de ações do usuário ou do sistema que ocorrem nos objetos do vSphere. Há três tipos de eventos: Informações, Aviso e Erro. Alarmes são notificações ativadas em resposta a um evento, conjunto de condições ou estado de um objeto do vSphere. Os alarmes têm os níveis de severidade a seguir: Normal - verde, Aviso - amarelo e Alerta - vermelho. Ações de alarme são operações que ocorrem em resposta a um alarme, por exemplo, uma notificação por e-mail é enviada. Para obter informações sobre como é possível configurar e usar o monitoramento do vSphere, consulte [Monitorando eventos, alarmes e ações automatizadas](https://docs.vmware.com/en/VMware-vSphere/6.5/com.vmware.vsphere.monitoring.doc/GUID-9272E3B2-6A7F-427B-994C-B15FF8CADC25.html){:new_window}. |

Para obter mais informações, consulte [Incluindo, visualizando e excluindo clusters para instâncias do vCenter Server](/docs/services/vmwaresolutions/services?topic=vmware-solutions-vc_addingviewingclusters#vc_addingviewingclusters){:new_window}. |
| Capacidade do vCenter Server | É possível ampliar ou reduzir a capacidade de sua instância do VMware vCenter Server de acordo com suas necessidades de negócios, incluindo ou removendo servidores do vSphere ESXi ou o armazenamento do sistema de arquivos de rede (NFS). Para obter mais informações, consulte [Expandindo e contraindo a capacidade para instâncias do servidor do vCenter](/docs/services/vmwaresolutions?topic=vmware-solutions-vc_addingremovingservers#vc_addingremovingservers){:new_window}. |
| Serviços da instância do vCenter Server | Depois que a instância do vCenter Server é implementada, é possível incluir serviços adicionais, como uma solução de recuperação de desastre, de severidade ou de backup. Quando você não precisar mais desses serviços, será possível removê-los de suas instâncias. Para obter mais informações, veja [Pedindo, visualizando e removendo serviços para instâncias do vCenter Server](/docs/services/vmwaresolutions?topic=vmware-solutions-vc_addingremovingservices#vc_addingremovingservices){:new_window}. |
| Atualização de instâncias do vCenter Server | Para obter informações sobre como aplicar atualizações a instâncias do vCenter Server, consulte [Aplicando as atualizações do componente de gerenciamento da IBM às instâncias do vCenter Server](/docs/services/vmwaresolutions?topic=vmware-solutions-vc_applyingupdates#vc_applyingupdates). Para obter informações sobre a aplicação manual de atualizações do produto VMware por administradores do sistema, consulte [Visão geral do vCenter Server on IBM Cloud with Hybridity Bundle](/docs/services/vmwaresolutions?topic=vmware-solutions-vcs-hybridity-intro#vcs-hybridity-intro){:new_window}. |
| Configuração multissite | O recurso de configuração de vários sites usa uma topologia de hub e spoke com um site primário e um máximo de sete sites secundários. Para obter informações sobre a definição de uma configuração multissite para instâncias do vCenter Server on {{site.data.keyword.cloud_notm}}, consulte [Configuração multissite para instâncias do vCenter Server on IBM Cloud](/docs/services/vmwaresolutions?topic=vmware-solutions-vc_multisite#vc_multisite){:new_window}. Para obter informações sobre como os administradores do sistema podem usar o VUM em uma Instância do vCenter Server, consulte [Introdução ao VMware Update Manager](/docs/services/vmwaresolutions?topic=vmware-solutions-vum-intro#vum-intro){:new_window}. |
| Backup dos principais componentes da instância do vCenter Server | Os administradores do seu sistema são responsáveis pela configuração, pelo gerenciamento e pelo monitoramento de todos os componentes de software da instância do vCenter Server, incluindo o backup e a disponibilidade da infraestrutura de gerenciamento e das cargas de trabalho. Como parte da solução, é possível implementar os serviços complementares IBM Spectrum Protect Plus on {{site.data.keyword.cloud_notm}} ou Veeam on {{site.data.keyword.cloud_notm}} opcionalmente. O Veeam e o IBM Spectrum Protect Plus podem ajudar a satisfazer o requisito para fazer backup dos componentes de gerenciamento. Para obter informações sobre os mecanismos de backup e os componentes principais a terem o backup realizado, consulte [Fazendo backup de componentes](/docs/services/vmwaresolutions/archiref/solution?topic=vmware-solutions-solution_backingup). |
