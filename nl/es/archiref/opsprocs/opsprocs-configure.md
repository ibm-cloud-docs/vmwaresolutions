# Tareas de configuración
{: #opsprocs-configure}

Una vez suministrada la instancia de vCenter Server, hay varias tareas de configuración que los administradores del sistema pueden ejecutar para adaptar el entorno inicial a las necesidades de su empresa, y para responder a las futuras solicitudes de servicio. 

Estas tareas de configuración son:

* Instrucciones para instancias y clústeres
* Procedimientos de máquinas virtuales (VM)
* Procedimientos de vCenter
* Procedimientos de host de vSphere ESXi
* Procedimientos de almacenamiento
* Procedimientos de red

## Instrucciones para instancias y clústeres
{: #opsprocs-configure-instance}

Tabla 1. Instancias y clústeres de vCenter Server

| Título | Descripción |
|---|---|
| Convenio de denominación  |  Una de las primeras tareas consiste en adoptar un convenio de denominación para la instancia de vCenter Server. Puede ampliar el convenio de denominación existente utilizado por la organización o crear uno. Para obtener más información, consulte [Convenio de denominación en VMware Validated Design 4.1](https://docs.vmware.com/en/VMware-Validated-Design/4.1/com.vmware.vvd.sddc-upgrade.doc/GUID-FCA126F1-E429-454E-B3BA-226525742DC1.html){:new_window}. | Conectar con la instancia de vCenter Server  | Una vez haya suministrado una instancia, conecte con to vCenter. Inicialmente, hay dos métodos para hacerlo. Puede desplegar un servidor de "jumbox" para poder conectarse al jumbox utilizando Internet y, a continuación, del jumbox a vCenter utilizando la red privada. El segundo método es utilizar la VPN de {{site.data.keyword.cloud_notm}}, que es una conexión VPN desde su ubicación a la red privada de {{site.data.keyword.cloud_notm}}; esta opción permite la gestión y el acceso de vCenter fuera de banda. Para obtener más información, consulte [Iniciación a las redes privadas virtuales (VPN)](/docs/infrastructure/iaas-vpn?topic=VPN-getting-started). |
| Configurar la resolución de nombres |  Para utilizar las características de vCenter, añada la resolución de nombres para los componentes de vCenter Server a la estación de trabajo o al jumbox, según el método de conexión. El archivo /etc/hosts o c:\Windows\System32\Drivers\etc\hosts requiere entradas para los hosts de Platform Services Controller (PSC), y vSphere ESXi. Para obtener más información, consulte [Despliegue de un archivo OVF utilizando el cliente web de VMware vSphere](/docs/services/vmwaresolutions?topic=vmware-solutions-trbl_deploy_ovf#trbl_deploy_ovf){:new_window}. |
| RBAC | Para obtener información sobre los privilegios necesarios para las tareas comunes en el entorno de vSphere, consulte [Privilegios necesarios para la realización de tareas comunes](https://docs.vmware.com/en/VMware-vSphere/6.7/com.vmware.vsphere.vm_admin.doc/GUID-4D0F8E63-2961-4B71-B365-BBFA24673FDB.html){:new_window}. |
| Utilización del Certificado firmado de Autoridad certificadora personalizado |  Consulte [Sustitución de un certificado SSL de la máquina vSphere 6.x por un certificado firmado de entidad emisora de certificados personalizado (2112277)](https://kb.vmware.com/s/article/2112277){:new_window}. |
| Configurar la supervisión en vSphere  | vSphere incluye un subsistema configurable por el usuario de sucesos y alarmas, que hace un seguimiento de los sucesos de los componentes de vSphere y almacena los datos en archivos de registro y en la base de datos de vCenter. Los sucesos son registros de acciones de usuario o del sistema que se producen en los objetos de vSphere. Hay tres tipos de sucesos: Información, Aviso y Error. Las alarmas son notificaciones que se activan en respuesta a un suceso, a un conjunto de condiciones o al estado de un objeto de vSphere. Las alarmas tienen los siguientes niveles de gravedad: Normal – verde, Aviso – amarillo y Alerta – rojo. Las acciones de alarma son operaciones que se producen en respuesta a una alarma; p. ej. enviar una notificación de correo electrónico. Para obtener información sobre cómo puede configurar y utilizar la supervisión de vSphere, consulte [Supervisión de sucesos, alarmas y acciones automatizadas](https://docs.vmware.com/en/VMware-vSphere/6.5/com.vmware.vsphere.monitoring.doc/GUID-9272E3B2-6A7F-427B-994C-B15FF8CADC25.html){:new_window}. |
| Añadir, ver y suprimir clústeres | La instancia de vCenter Server se ha desplegado con un clúster de hosts de vSphere ESXi. Para obtener más información, consulte [Adición, visualización y supresión de clústeres para instancias de vCenter Server](/docs/services/vmwaresolutions/services?topic=vmware-solutions-vc_addingviewingclusters#vc_addingviewingclusters){:new_window}. |
| Capacidad de vCenter Server | Puede ampliar o reducir la capacidad de la instancia de VMware vCenter Server según sus necesidades empresariales añadiendo o eliminando servidores de vSphere ESXi o almacenamiento del sistema de archivos de red (NFS). Para obtener más información, consulte [Ampliación y reducción de capacidad para instancias de vCenter Server](/docs/services/vmwaresolutions?topic=vmware-solutions-vc_addingremovingservers#vc_addingremovingservers){:new_window}. |
| Servicios de instancia de vCenter Server | Después de desplegar la instancia de vCenter Server, puede añadir servicios de adición, como por ejemplo, una solución para recuperación tras desastre, para errores graves o de copia de seguridad. Cuando ya no necesite estos servicios, puede eliminarlos de sus instancias. Para obtener más información, consulte [Solicitud, visualización y eliminación de servicios para instancias de vCenter Server](/docs/services/vmwaresolutions?topic=vmware-solutions-vc_addingremovingservices#vc_addingremovingservices){:new_window}. |
| Actualización de instancias de vCenter Server | Para obtener información sobre cómo aplicar actualizaciones a instancias de vCenter Server, consulte [Aplicación de actualizaciones de componentes de gestión de IBM a instancias de vCenter Server](/docs/services/vmwaresolutions?topic=vmware-solutions-vc_applyingupdates#vc_applyingupdates). Para obtener información sobre cómo los administradores del sistema pueden aplicar actualizaciones de productos de VMware manualmente, consulte [Visión general de vCenter Server on IBM Cloud con el paquete híbrido (Hybridity Bundle)](/docs/services/vmwaresolutions?topic=vmware-solutions-vcs-hybridity-intro#vcs-hybridity-intro){:new_window}. |
| Configuración de varios sitios | La característica de configuración de varios sitios utiliza una topología de estrella ("hub and spoke") con un sitio primario y un máximo de siete sitios secundarios. Para obtener información sobre cómo configurar una configuración de varios sitios para instancias de vCenter Server on {{site.data.keyword.cloud_notm}}, consulte [Configuración de varios sitios para instancias de vCenter Server on IBM Cloud](/docs/services/vmwaresolutions?topic=vmware-solutions-vc_multisite#vc_multisite){:new_window}. | Uso de VMware Update Manager | Una vez desplegados, los productos de VMware los actualizan los administradores del sistema. Para obtener información sobre cómo los administradores del sistema pueden utilizar VUM en una instancia de vCenter Server, consulte [Introducción a VMware Update Manager](/docs/services/vmwaresolutions?topic=vmware-solutions-vum-intro#vum-intro){:new_window}. |
| Copia de seguridad de componentes clave de una instancia de vCenter Server | Los administradores del sistema son responsables de la configuración, la gestión y la supervisión de todos los componentes de software de la instancia de vCenter Server, incluyendo la copia de seguridad y la disponibilidad de la infraestructura de gestión y las cargas de trabajo. Como parte de la solución, puede desplegar opcionalmente los servicios complementarios IBM Spectrum Protect Plus on {{site.data.keyword.cloud_notm}} o Veeam on {{site.data.keyword.cloud_notm}}. Veeam e IBM Spectrum Protect Plus pueden ayudar a satisfacer el requisito de realizar copia de seguridad de los componentes de gestión. Para obtener información sobre los mecanismos de copia de seguridad y los componentes clave de los que se debe hacer copia de seguridad, consulte [Copia de seguridad de los componentes](/docs/services/vmwaresolutions/archiref/solution?topic=vmware-solutions-solution_backingup). |
