# 訂購 Caveonix RiskForesight on IBM Cloud
{: #caveonix_ordering}

您可以在訂購內含 Caveonix RiskForesight on {{site.data.keyword.cloud}} 服務的新 VMware vCenter Server 服務時訂購該新服務，也可以將該服務新增至現有實例。

## 為新實例訂購 Caveonix RiskForesight on IBM Cloud
{: #caveonix_ordering-new}

若要訂購內含 Caveonix RiskForesight on {{site.data.keyword.cloud_notm}} 的新 VMware vCenter Server on {{site.data.keyword.cloud_notm}} 實例，請在您從 {{site.data.keyword.vmwaresolutions_short}} 主控台訂購該實例時，在**服務**區段中選取 **Caveonix RiskForesight on IBM Cloud**。

## 為現有實例訂購 Caveonix RiskForesight on IBM Cloud
{: #caveonix_ordering-existing}

若要將 Caveonix RiskForesight on {{site.data.keyword.cloud_notm}} 服務新增至現有 VMware vCenter Server on {{site.data.keyword.cloud_notm}} 實例，請檢視您要對其新增服務的實例，接著按一下左導覽窗格中的**服務**，然後按一下**新增**。

## Caveonix RiskForesight on IBM Cloud 配置
{: #caveonix_ordering-config}

當您訂購 Caveonix RiskForesight on {{site.data.keyword.cloud_notm}} 服務時，請提供下列設定。

### 授權名稱
{: #caveonix_ordering-config-license-name}

指定授權的唯一名稱。

### 授權附註
{: #caveonix_ordering-config-license-notes}

輸入服務實例的附註。之後，您可以在授權詳細資料頁面上編輯附註。

### 要授權的 VM 數目
{: #caveonix_ordering-config-vms}

選取要授權的 VM 數目（10 到最多 1000 份 VM 授權，增量為 10）。至少需要 10 部 VM 以用於授權管理。

訂購 Caveonix RiskForesight on {{site.data.keyword.cloud_notm}} 服務之後，如果您需要比訂購中所指定更多授權的 VM，您可以藉由訂購更大的授權，並將該授權複製至 Caveonix RiskForesight 主控台，然後再從 {{site.data.keyword.vmwaresolutions_short}} 主控台刪除之前的授權，來取代您的 Caveonix RiskForesight 授權。如需相關資訊，請參閱[訂購 Caveonix RiskForesight 授權的程序](/docs/services/vmwaresolutions?topic=vmware-solutions-caveonix_license_ordering#caveonix_license_ordering-procedure)和[刪除 Caveonix RiskForesight 授權的程序](/docs/services/vmwaresolutions?topic=vmware-solutions-caveonix_license_managing#caveonix_license_managing_procedure-delete)。
{:note}
